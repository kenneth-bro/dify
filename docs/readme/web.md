
# 目录结构介绍
```
husky: 通常用于存放Git钩子脚本，Husky是一个流行的选择，用于自动化Git钩子。
.idea: IntelliJ IDEA的工作空间配置文件夹。
next: 可能与Next.js框架相关，存放框架特定的配置或文件。
.swc: 与Swc编译器相关的缓存或配置文件。
.vscode: VSCode编辑器的配置文件夹。
app: 应用程序的主要代码存放位置。
assets: 存放静态资源如图片、字体等。
bin: 存储可执行脚本或二进制文件。
config: 配置文件存放位置。
context: 可能用于存放上下文相关信息或模块。
docker: Docker相关配置或脚本。
hooks: 自定义钩子或其他逻辑。
i18n: 国际化和本地化资源。
models: 数据模型定义。
node_modules: Node.js项目的依赖库。
library: 可能是自定义库或第三方库的存放位置。
public: 公开的静态文件，例如HTML页面。
service: 服务端代码或API接口实现。
themes: 主题相关文件。
types: TypeScript类型定义文件。
utils: 工具函数或通用功能模块。
.dockerignore: Docker构建时忽略的文件列表。
.editorconfig: 编辑器配置文件，确保代码风格一致性。
.env.example: 环境变量示例文件。
.env.local: 本地环境变量配置。
.eslintignore: ESLint忽略的文件列表。
.eslintrc.json: ESLint配置文件。
gitignore: Git版本控制忽略的文件列表。
Dockerfile: Docker镜像构建脚本。
global.d.ts: TypeScript全局声明文件。
jest.config.ts: Jest测试框架配置文件。
```
